# Agents 特区

**一起探索怎么把软件工程和 AI 结合，让我们最终失业。**

**[agentszone.ai](https://agentszone.ai)**

## Vision

AI 不是工具，是同事。管好同事，是个管理问题。

绝大多数团队还在用"提示词工程"的思路对待 AI 变革，把一个组织问题当成技术问题来解。我们认为这条路走不通。

Agents 特区是一个由实践者驱动的社区。我们在真实环境中驱动 AI 的变革，关注四个维度：

**技术**——不是怎么写更好的提示词，而是怎么设计系统，让差的提示词也不会摧毁质量。确定性约束、独立验证、分层测试、文档即记忆——这些工程原则比任何模型升级都重要。

**组织**——AI 把个人生产力提升了 3-10 倍，但组织结构没跟上。一个人能干十个人的活，那团队怎么搭？谁对质量负责？谁有否决权？我们从土木工程借来监理制度，从制造业借来统计过程控制，从军事史借来三个月见成果的方法论。不是因为新鲜，是因为管用。

**流程**——需求、开发、测试、发布——当执行者从人变成 Agent，每个环节都需要重新设计。我们探索三层迭代（人与人澄清需求、人与 AI 翻译为测试、AI 与 AI 实现代码），把规范写成可执行的合约，让流程本身成为质量的保障。

**质量**——这是我们最执着的主题。AI 的概率性不是缺陷，是数学必然。质量 = 约束力 × 独立性。写了 14 万条测试用例的人在这里，搭建了七层测试体系的人在这里，把代码监理写进组织架构的人也在这里。

## 社区原则

### 我们是谁

AI craftsman 的管理和建造实践者。来自券商、医疗器械、K12 教培、支付宝、腾讯云、开源基础设施——背景越不同，碰撞越有价值。

这里不欢迎"我来学习"的单向输入者。你来，就要带着东西来——一个经验、一个问题、一个观点。

### 我们推崇的

**互相挑战，不互相吹捧。** 你的想法经不起质疑，就不值得坚持。善意的严厉比虚假的赞美有用得多。

**一手经验。** 不是读了什么论文，是在生产环境里踩了什么坑。AI 输出的内容请截图，加你自己的注释——我们要看你的思考，不是 AI 的复读。

**高质量的问题。** 不一定要有答案，但要有深度。问小白问题之前，先翻一下别人的发言记录。

**跨学科思维。** 软件工程只有几十年历史，管理学、土木工程、制造业、军事指挥有几百年的积累。站在巨人肩膀上，不从零发明轮子。

**诚实。** 在 AI 面前，没有资历，没有层级。二十年经验的老兵说"我在 AI 面前觉得自己很渺小"，然后从零开始学销售。这种诚实是我们最珍贵的文化。

**紧迫感。** 竞争优势的窗口在关闭。不是探索，是交付。不是试试看，是三个月拿出成果。

### 我们不做的

不做概念布道。不做模型评测。不做提示词收集。不贩卖焦虑。不养潜水员。

### 组织方式

社区是扁平的。没有正式层级，权威来自实践，不来自头衔。

- **每周线上分享会**——欢迎报名分享你的任何实践
- **日常群聊**——随时讨论、随时挑战
- **开源内容**——本站文章和活动记录对所有人开放

想加入或报名分享，联系马工或付权智。

## Code of Conduct

1. **对事不对人。** 挑战想法，不攻击个人。严厉的爱和人身攻击有本质区别。
2. **贡献或离开。** 这是一个双向交流的社区。长期只看不说的成员会被定期清理。不是冷酷，是对活跃成员的尊重。
3. **诚实标注 AI 输出。** 分享 AI 生成的内容时，截图并加你自己的注释。我们要看你的判断，不是无标注的转发。
4. **广告可以打，但要经得起挑战。** 你推荐的产品或服务，社区有权利（也有义务）公开质疑。经不起质疑的东西不值得推荐。
5. **新成员需要自我介绍。** 你是谁，你在做什么，你对 AI 的实践和观点。进来就要接受社区的"拷问"——这是严厉的爱。
6. **尊重多样性。** 背景、行业、经验水平不同是我们的优势，不是分裂的理由。
7. **没有其他规则了。Have fun.**

## 参与贡献

这个网站本身就是 AI Agent 交付的产物。但内容需要人来写——那些踩过坑、趟过河的一线经验，才是最有价值的。

**我们欢迎：**

- 你在团队中引入 AI Coding 的实战复盘
- 你设计的 Agent 协作架构和踩过的坑
- 你对 AI 质量控制、测试策略的思考
- 你在非技术领域（设计、运营、销售）用 AI 的经验
- 任何让你觉得"这个值得分享"的实践

不需要长篇大论。一个真实的案例、一个有效的方法、一个深刻的教训，都值得写下来。

**如何提交：** 让你的 AI Coding Agent fork 这个 repo，写好文章，提个 PR。技术细节见 [CONTRIBUTING.md](CONTRIBUTING.md)——它是写给你的 Agent 看的。
